# Основная часть
В ходе данной лабораторной работы я реализовал автоматизированный ETL-пайплайн с использованием фреймворка Apache Spark для трансформации данных и создания аналитических витрин ClickHouse. Выполнение работы проходило в несколько этапов:
1) С помощью docker-compose.yml была поднята среда, включающая реляционную БД PostgreSQL (аналогичная БД из прошлой лабораторной работы), Apache Spark (вычислительный движок) и NoSQL БД ClickHouse (аналитическая витрина). Настроены Healthchecks для обеспечения строгого порядка запуска сервисов (сначала поднимается БД, а затем производится распределение данных и создание витрин ClickHouse с помощью Apache Spark).
2) На языке Python c использованием фреймворка PySpark был написан код, который извлекает 10 000 записей из исходной таблицы PostgreSQL, выполняет их очистку (обработка пробелов, типов данных и дубликатов) и распределяет по нормализованной модели «Звезда». Особое внимание уделено сохранению целостности данных и связей между измерениями. Итоговое заполнение данных модели «Звезда» проходит все тесты на целостность данных, написанные в прошлой лабораторной работе.
3) На основе модели «Звезда» и задании лабораторной работы были разработаны 6 витрин данных, включающие 18 детализированных отчетов. Отчеты охватывают анализ продаж по продуктам, клиентам, поставщикам, магазинам, а также анализ сезонности и качества продукции (какие отчёты конкретно были реализованы написано в README.md).
4) Организована выгрузка сформированных витрин в ClickHouse через JDBC-интерфейс. Реализована обработка специфических требований NoSQL базы, таких как отсутствие поддержки NULL (было заменено на N/A) в строковых колонках и оптимизация записи.

# Заключение
Выполнение данной работы позволило мне освоить Apache Spark — один из ключевых инструментов в Big Data. Я научился строить масштабируемые ETL-пайплайны, которые связывают различные типы баз данных (SQL и NoSQL). Практическое применение Spark для трансформации данных показало преимущество распределенных вычислений при работе с нормализованными моделями. Кроме того, опыт использования ClickHouse в качестве аналитического слоя закрепил понимание современной архитектуры данных, где хранение и быстрая аналитика разделены между специализированными системами. Эти навыки являются критически важными для роли Data-инженера и разработки высоконагруженных систем.
